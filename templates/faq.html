🐍 Python Programming FAQs

▼ What is the difference between list and tuple?

Lists are mutable (can be modified) while tuples are immutable (cannot be changed).

  • Lists use [], tuples use ()
  • Lists slower but flexible, tuples faster and fixed
  • Tuples can be dictionary keys, lists cannot
  • Use lists for changing data, tuples for fixed collections
▼ What are list comprehensions and when to use them?

Concise way to create lists. Example: [x*2 for x in range(10)]

  • Can include conditions: [x for x in range(10) if x % 2 == 0]
  • Faster and more Pythonic than loops
  • Use for transforming or filtering lists
  • Improves code readability
▼ What are decorators and how do they work?

Functions that modify or enhance other functions without changing them permanently.

  • Defined with @ symbol before function
  • Common uses: logging, authentication, caching
  • Wrap a function and modify its behavior
  • Example: @decorator def my_function(): pass
▼ What is the GIL (Global Interpreter Lock)?

Mutex allowing only one thread to execute Python bytecode at a time.

  • Prevents true parallel execution in multi-threaded programs
  • Doesn't affect multi-process programs
  • Necessary for CPython memory management
  • Limitation for CPU-bound multi-threaded applications

☕ Java Programming FAQs

▼ Difference between abstract classes and interfaces?

Both for abstraction but with key differences:

  • Abstract classes have concrete methods, interfaces cannot (Java 8)
  • Extends for abstract, implements for interfaces
  • One abstract class, multiple interfaces allowed
  • Abstract classes have state, interfaces generally don't
▼ Types of exception handling in Java?

Multiple ways to handle exceptions:

  • Checked: Must catch or declare (IOException, SQLException)
  • Unchecked: Don't need to catch (NullPointerException)
  • Try-Catch-Finally: Catch exceptions, finally always executes
  • Try-with-resources: Auto-closes resources (Java 7+)
▼ Difference between == and .equals()?

Both compare but work differently:

  • == compares object references/memory addresses
  • .equals() compares actual content/value
  • For Strings, always use .equals() for content comparison
  • == checks if two objects are the same instance
▼ What is Collections Framework?

Provides classes and interfaces for storing and manipulating groups of objects:

  • List: Ordered, duplicates allowed (ArrayList, LinkedList)
  • Set: Unordered, no duplicates (HashSet, TreeSet)
  • Map: Key-value pairs (HashMap, TreeMap)
  • Each has different performance for different operations

🗄️ SQL & Databases FAQs

▼ Different types of JOINs in SQL?

JOINs combine rows from multiple tables:

  • INNER JOIN: Rows existing in both tables
  • LEFT JOIN: All left table rows + matching right rows
  • RIGHT JOIN: All right table rows + matching left rows
  • FULL JOIN: All rows from both tables
▼ What is database normalization?

Organizes data to minimize redundancy and dependency:

  • 1NF: Atomic values, no repeating groups
  • 2NF: Non-key attributes depend on entire primary key
  • 3NF: No transitive dependencies between non-key attributes
  • Reduces duplication and improves data integrity
▼ WHERE vs HAVING clauses?

Both filter data but at different stages:

  • WHERE: Filters rows before grouping
  • HAVING: Filters groups after aggregation
  • WHERE can't use aggregate functions, HAVING can
  • Example: WHERE age > 25 vs HAVING COUNT(*) > 5
▼ What are indexes and their importance?

Data structures that speed up data retrieval:

  • Create sorted copy for faster lookup
  • Reduce I/O operations needed to find data
  • Slow down INSERT, UPDATE, DELETE operations
  • Best for frequently searched columns

⚙️ C Programming FAQs

▼ What are pointers and how do they work?

Pointers store memory addresses of variables:

  • Declared with *: int *ptr;
  • & operator gets address: ptr = &x;
  • * operator dereferences: printf("%d", *ptr);
  • Essential for dynamic memory and complex structures
▼ Stack vs Heap memory?

Both memory regions but work differently:

  • Stack: Local variables, function parameters, return addresses
  • Heap: Dynamically allocated, larger but slower
  • Stack memory auto-freed when variable goes out of scope
  • Heap memory must be manually freed using free()
▼ Dynamic memory allocation and when to use?

Allocate memory at runtime:

  • malloc(): Allocates memory, returns void pointer
  • calloc(): Allocates and initializes to zero
  • free(): Deallocates to prevent leaks
  • Use when size unknown at compile time or need large data
▼ Structures vs Arrays?

Structures group different data types together:

  • Array: Collection of same data type
  • Structure: Collection of different data types
  • Access structure members with . or -> operator
  • Useful for creating complex data with related information

🤖 AI & Machine Learning FAQs

▼ AI vs Machine Learning vs Deep Learning?

Related terms with different scopes:

  • AI: Broad field of creating intelligent machines
  • ML: Subset of AI where systems learn from data
  • DL: Subset of ML using neural networks with many layers
  • Each builds upon previous one in scope and capability
▼ What are neural networks?

Mimic biological neurons for pattern recognition:

  • Neurons: Processing units with inputs and outputs
  • Weights: Parameters determining neuron importance
  • Activation Functions: Introduce non-linearity (ReLU, Sigmoid)
  • Layers: Input, hidden (process), and output layers
▼ What is backpropagation?

Algorithm for training neural networks:

  • Forward pass: Compute predictions through network
  • Calculate loss: Compare predictions with actual values
  • Backward pass: Propagate error back through layers
  • Update weights: Adjust to minimize loss
▼ What is overfitting and how to prevent?

When model memorizes training data instead of learning patterns:

  • Model performs well on training but poorly on new data
  • Prevention: Use regularization (L1/L2), dropout, early stopping
  • Cross-validation helps detect overfitting
  • Use simpler models when possible

🌐 Full Stack Development FAQs

▼ GET vs POST requests?

Both HTTP methods but serve different purposes:

  • GET: Retrieves data, visible in URL, has size limit
  • POST: Sends data, in body, no visible URL
  • GET is idempotent, POST can modify data
  • Use GET for retrieving, POST for creating/modifying
▼ What is REST API?

Architectural style for web services:

  • Client-Server: Separate concerns
  • Stateless: Each request self-contained
  • Resources: Data as resources with URIs
  • Use: GET (read), POST (create), PUT (update), DELETE
▼ Synchronous vs Asynchronous?

Handle operations differently in timing:

  • Sync: Code executes line by line, waits
  • Async: Operations execute without waiting
  • Async improves performance for I/O-bound operations
  • JavaScript: async/await, callbacks, Promises
▼ Authentication vs Authorization?

Security concepts serving different purposes:

  • Auth: Verifies who the user is
  • Authz: Determines what resources user can access
  • Methods: JWT tokens, sessions, OAuth 2.0
  • Always use HTTPS to protect credentials
